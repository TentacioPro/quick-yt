# 14 — Expo SDK 54 Upgrade Learnings

## Discovered Gotchas & Bugs Resolved
- **React Navigation Crash (`Cannot read property 'bold' of undefined`)**: Found an obscure edge case where overriding the `<NavigationContainer>` theme with a partial object (only specifying `colors`) strips the `fonts` object from the context. React Navigation's internal Header component blindly expects `theme.fonts.bold` (or similar) to exist. The stack trace was completely opaque since the crash was reported as `TypeError: Cannot read property 'bold' of undefined` within a bundled JavaScript payload at runtime. The resolution was extracting `DarkTheme` directly from `@react-navigation/native` and spreading it into the custom override.
- **Expo SDK 54 Worklet Conflicts**: Running `npx expo install --fix` identified issues with `@react-native/worklets`. The automated fix can loop or lock dependencies poorly. Manually resolving the version in `package.json` to `0.5.1` and utilizing `pnpm install` successfully cleared the module conflict.
- **Drawer Props Routing**: Discovered that if a screen is launched directly from the Drawer (e.g. `VideoDetail`), it lacks any params unless strictly provided. The `VideoDetailScreen` originally crashed because it assumed the `video` prop would always be passed directly as a React component prop, rather than parsing `route.params`. The screen was modified to handle empty states gracefully and correctly read from the `route.params` context.
